# 이진 탐색

## 변경 전

```python
start = 0
end = len(lst)-1

while start <= end:
  ...
  elif lst[mid] < target:
    start = mid+1

  else:
    end = mid-1
```
지금 까지 풀던 프레임이다. 
하지만 `start <= end` 이 부분에 대해 확실함이 없었다. 
문제를 풀때마다 예외가 가끔식 발생했다.

## 변경 후 
```python

start = 0
end = len(lst)  # 변경1

while start < end: # 변경2
  ...
  elif lst[mid] < target
    start = mid+1

  else:
    end = mid # 변경3
```
3개의 변경 사항이 생겼다.

- 일반적인 풀이:	start < end
- (파라메트릭 서치)정답을 만족하는 값중 가장 큰 수:	start < end
- (파라메트릭 서치)정답을 만족하는 가장 작은 수:	start <= end
￼
