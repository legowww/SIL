## 프로그레머스-자바 책 참조 
```python

start = 0
end = len(lst)  # 변경1

while start < end: # 변경2
  ...
  elif lst[mid] < target
    start = mid+1

  else:
    end = mid # 변경2
```

### 변경1
- 일반적인 풀이:	[start, end)
- (파라메트릭 서치)정답을 만족하는 값중 가장 큰 수:	[start, end)
- (파라메트릭 서치)정답을 만족하는 가장 작은 수:	[start, end]
￼
### 변경2
이거는 취향 차이이므로 알아서 하자 개인적으로 이전 방식이 나은듯하다.
파라메트릭 서치 때는 정답을 기록하는 변수 answer 등을 사용하여 값을 기록한다.
