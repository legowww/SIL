DBMS란?
-> 데이터베이스 내 데이터에 접근하도록 도와준느 시스템

DBMS 구조
-> MySQL엔진과 스토리지 엔진으로 이루어져 있습니다.
   MySQL엔진은 클라이언트와 커넥션을 유지하며, SQL 문장을 분석하는 파서, 전처리기, 옵티마이저, 실행엔진
   등을 통해 DBMS의 두뇌와 같은 일을 합니다. 스토리지 엔진은 실제 데이터를 디스크에 쓰거나 읽는 역할을 수행합니다. 

질의처리기 작동 순서
-> 파서(문법검사), 전처리기(테이블 및 컬럼 매핑), 옵티마이저(최적화 된 실행계획 생성), 
	실행엔진(실행계획을 통해 데이터 가져옮), 스토리지 엔진(OLTP 환경은 InnoDB)


DDL, DML, DCL?
-> DDL 은 테이블 정의를 위한 언어
   DML 은 레코드에 대한 삽입,삭제,수정,조회 등을 수행하는 언어	
   DCL 은 데이터베이스를 제어하는 언어 COMMIT, ROLLBACK

   DELETE: 테이블에서 레코드를 삭제	
   TRUNCATE: 롤백 불가능하며, 인덱스 까지 삭제, 원하는 부분만 삭제 불가능, CREATE TABLE 상태로 돌아감
   DROP: 롤백 불가능하며, 인덱스 까지 삭제, 원하는 부분만 삭제 불가능, 테이블까지 완전히 제거

Key
-> 슈퍼키: 레코드를 유니크하게 식별할 수 있는 키의 set
   후보키: 레코드를 유니크하게 식별할 수 있는 키의 최소 set
   기본키: 후보키중 선택된 키
   외래키: 다른 테이블의 기본키를 참조하는 키
   

뷰
-> 임시적으로 생성되는 테이블, FROM 절에서 서브쿼리를 사용하는 것을 인라인 뷰라고 부릅니다.

스토어드 펑션
->  CREATEZ FUNCTION, 주로 계산에 사용, 트랜잭션 사용 불가능, 반환값 필수

스토어드 프로시저
-> CREATE PROCEDUTRE, 주로 비지니스 로직, 트랜잭션 사용 가능

트리거
-> 데이터에 변경이 생겼을 때, 자동적으로 실행되는 프로시저
   ex) nickname 컬럼이 변경됐을 때, 유저로그라는 테이블에 데이터를 추가하는 트리거 수행	

무결성
무결성 보장방법
정규화
정규형 필요 조건
함수적 종속성
SQL Injection
